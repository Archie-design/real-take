## 1. 進入管理後台（大會中樞）

### 1.1 開啟管理後台

1. 登入系統後，在主頁面右上角尋找「**大會中樞**」入口（或由大會指定路徑進入）
2. 輸入**管理員密碼**（請向大會確認）
3. 若密碼錯誤，系統顯示「密令錯誤，大會禁地不可擅闖」
4. 密碼正確後進入管理後台

### 1.2 後台頁籤結構

管理後台分為四個頁籤：

| 頁籤 | 說明 |
|------|------|
| 成員 | 名冊匯入、成員搜尋與編輯 |
| 任務 | 定課分值設定、週任務管理、臨時任務 |
| 審核 | 傳愛申請終審 |
| 系統 | 罰款設定、場次管理、志工密碼、活動紀錄 |

## 2. 成員管理

進入「**成員**」頁籤。

---

### 2.1 報名模式切換

控制誰可以進行系統報名：

| 模式 | 說明 |
|------|------|
| 🔐 名單驗證模式 | 只有在名冊內的手機號碼才能報名 |
| 🌐 自由報名模式 | 任何人皆可自行報名 |

**操作步驟：**
1. 點擊「**報名模式**」切換按鈕
2. 系統即時更新，不需另外儲存

> **建議：** 課程正式開始後，維持「名單驗證模式」以防止未授權人員進入。

---

### 2.2 批量匯入名冊（CSV 格式）

**使用時機：** 學員報名時、小隊分組調整時、補建成員資料時

**CSV 格式規範：**

```
手機,姓名,生日(YYYY-MM-DD),大隊,小隊,是否小隊長,是否大隊長
0912345678,王大明,1990-05-15,第一大隊,第一小隊,false,false
0922345678,李小花,1988-03-20,第一大隊,第一小隊,true,false
0932345678,張志強,1985-07-10,第一大隊,,false,true
```

**欄位說明：**
- `手機`：台灣手機號碼，作為唯一識別碼（必填）
- `姓名`：顯示名稱（必填）
- `生日`：YYYY-MM-DD 格式（可留空）
- `大隊`：大隊名稱（可留空）
- `小隊`：小隊名稱（可留空）
- `是否小隊長`：`true` 或 `false`
- `是否大隊長`：`true` 或 `false`

**匯入步驟：**
1. 準備好 CSV 資料（可用 Excel 或 Google Sheets 匯出，注意編碼為 UTF-8）
2. 在「名冊管理」區塊，將 CSV 內容貼入文字方塊
3. 點擊「**📥 批量匯入名冊**」
4. 系統顯示成功匯入筆數

> **重要說明：**
> - 手機號碼為主鍵（Primary Key），相同手機號碼的資料會**覆蓋更新**
> - 空白列或無手機號碼的列會自動略過
> - 若該手機號碼對應的學員已建立帳號，匯入後會自動同步大隊、小隊、角色等資訊

---

### 2.3 搜尋與編輯成員

**搜尋成員：**
1. 在搜尋欄輸入：姓名、Email、大隊名稱或小隊名稱
2. 列表即時篩選

**編輯成員資料：**
1. 找到目標成員，點擊「**編輯**」按鈕
2. 可修改：
   - 大隊名稱（下拉自動補全）
   - 小隊名稱（下拉自動補全）
   - 角色：一般學員 / 小隊長 / 大隊長
3. 點擊「**儲存**」

> **注意：** 每個大隊/小隊只能有一位小隊長或大隊長。設定新角色時，系統會自動移除舊有的相同層級角色標記。

## 3. 任務管理

進入「**任務**」頁籤。

---

### 3.1 定課分值覆寫

為特定任務設定不同於預設值的加分。

**操作步驟：**
1. 在任務列表中找到目標任務
2. 在「分值」輸入欄中填入新分數（**留空** = 沿用預設值）
3. 所有調整完成後，點擊「**儲存定課設定**」

**各任務預設分值參考：**

| 任務ID | 名稱 | 預設分數 |
|--------|------|---------|
| q1 | 體運定課 | 1,000 |
| q1_dawn | 破曉打拳 | 2,000 |
| r1 | 關係定課 | 2,000（每人） |
| q2–q22 | 各任意定課 | 1,000 |
| a1 | 主創對談（天使通話） | 500 |
| w1 | 精彩片段分享 | 1,000 |
| w2 | 讚賞合製夥伴 | 1,000 |
| w3 | 劇組首映會 | 5,000 |
| w4 | 劇組會議 | 3,000 |

---

### 3.2 停用 / 啟用定課

**停用定課：**
1. 在任務列表，取消勾選目標任務前的核取方塊
2. 點擊「**儲存定課設定**」
3. 停用後：學員端該任務顯示為灰色，無法打卡

**啟用定課：**
1. 勾選目標任務的核取方塊
2. 點擊「**儲存定課設定**」

> **提醒：** 停用某個定課不會刪除已累積的打卡紀錄，只是暫時禁止新打卡。

---

### 3.3 臨時加分任務管理

用於設定特殊活動或加分副本（例如：特殊節慶活動、補充任務等）。

**新增臨時任務：**
1. 在「臨時加分任務管理」區塊填寫：
   - 任務標題（必填）
   - 副標題（選填）
   - 說明（選填）
   - 加分金額（NT，必須大於 0）
2. 點擊「**新增**」

**啟停臨時任務：**
- 在列表中點擊對應任務的 **✓（啟用）/ ×（停用）** 切換按鈕
- 停用後學員端不顯示該任務；啟用後立即可見

**刪除臨時任務：**
- 點擊列表中的垃圾桶圖示
- 刪除後不可復原

---

### 3.4 全服週任務自動抽籤

為所有「本週尚未抽籤」的小隊自動完成週任務抽籤。

**使用時機：** 每週一，確認所有小隊都有本週任務

**操作步驟：**
1. 點擊「**🎲 全服自動抽籤（為未抽劇組代選本週通告）**」
2. 系統回報：完成抽籤的小隊數量 / 跳過的小隊（已手動抽過）

> **說明：** 此操作等同幫所有還沒抽的小隊執行抽籤，不會重複抽已抽過的小隊。  
> 系統也會在每週一台灣時間 12:00 自動執行（見第 8 節）。

## 4. 電影推廣終審（審核 Tab）

進入「**審核**」頁籤。

此頁面顯示所有**已通過小隊長初審（squad_approved 狀態）**的申請，等待管理員最終審核。包含：
- **傳愛（b1/b2）**：訂金 / 完款引薦加分
- **加分任務（b3/b4/b7–b11）**：高階課程、心之使者、實體課程、會長交接、解圓夢、適應力挑戰、心之使者內訓

> **注意：** b5/b6（聯誼會）與 b12（三道菜）由小隊長審核即直接入帳，**不會出現**在此頁面。

**審核步驟：**
1. 查看申請資訊：
   - 學員姓名與所屬小隊
   - 申請類型與對象 / 說明
   - 小隊長備注（若有）
2. 輸入最終備注（選填）
3. 點擊「**核准**」→ 分數自動入帳，學員端狀態更新為「已核准」
4. 或點擊「**拒絕**」→ 申請狀態更新為「已拒絕」

> 核准後系統顯示「✅ 已核准入帳，電影推廣票房已發放！」  
> b2（完款）若同一對象已有核准的 b1（訂金），系統自動抵扣 1,000 分，實際入帳 2,000 分。  
> 所有審核動作均記錄於「管理活動紀錄」。

## 5. 系統設定

進入「**系統**」頁籤。

---

### 5.1 罰款設定

控制整個系統的罰款機制。

**設定欄位：**

| 欄位 | 說明 |
|------|------|
| 啟用罰款 | 勾選後罰款機制生效 |
| 每次罰款金額 | NT$（例如：50） |
| 適用任務 | 勾選哪些定課納入最低要求（如：q1、r1、w3 等） |
| 生效開始日期 | YYYY-MM-DD（留空 = 從活動開始計算） |
| 生效結束日期 | YYYY-MM-DD（留空 = 到活動結束）|

**操作步驟：**
1. 依需求填寫各欄位
2. 勾選/取消勾選適用的任務項目
3. 點擊「**儲存罰款設定**」

> **說明：** 設定儲存後，小隊長在執行「定課未達標結算」時，系統會依此設定計算罰款。

---

### 5.2 影展場次管理

管理首映曆中顯示的課程場次。

**新增場次：**
1. 填寫：
   - 場次 ID（唯一識別碼，英文/數字，建議使用 `screening_001` 格式）
   - 場次名稱
   - 日期（YYYY-MM-DD）
   - 時間（自由格式，如「19:00–21:40」）
   - 地點
2. 點擊「**新增**」

**編輯場次：**
1. 在場次列表找到目標場次
2. 點擊「**編輯**」
3. 修改欄位後點擊「**儲存**」，或「**取消**」放棄

**停用 / 啟用場次：**
- 切換「**啟用/停用**」開關
- 停用後學員端「首映曆」不顯示此場次，但資料保留

**刪除場次：**
- 點擊「**刪除**」按鈕
- 刪除後永久移除，報名資料可能受影響，請謹慎操作

---

### 5.3 志工掃碼密碼設定

設定工作人員（志工）進入課程 QR 掃碼模式所需的密碼。

**操作步驟：**
1. 在「志工掃碼授權」區塊輸入密碼
2. 點擊「**儲存**」

> **說明：** 此密碼僅用於「首映曆」頁面的「工作人員掃碼入口」，與管理員密碼完全獨立。  
> 建議設定容易記憶但不外流的密碼，並定期更換。

---

### 5.4 志工掃碼操作流程（給志工參考）

活動當天，志工掃碼操作：

1. 開啟系統網址，進入「首映曆」
2. 選擇對應場次
3. 點擊頁面底部「**工作人員掃碼入口**」
4. 輸入志工密碼，點擊「**進入場務掃描模式**」
5. 在掃描頁面：
   - 上方可切換場次
   - 點擊「開始掃碼」啟動相機
   - 掃描學員的入場 QR 碼
   - 頁面右側/下方顯示「已報到（X人）」名單，含報到時間
   - 掃描已報到的 QR 碼時，系統提示「已報到」（不重複記錄）

### 7.2 親證故事提交（LINE 自動解析）

學員在 LINE 群組發送符合格式的親證故事，Bot 自動解析並儲存。

**格式：**
```
#親證故事
姓名：[姓名]
日期：[YYYY-MM-DD]
類別：[類別]
內容：
[故事內容...]
```

也接受 `#親證` 作為觸發詞。

**Bot 動作：**
1. 解析欄位（姓名、日期、類別、內容）
2. 儲存至 Testimonies 資料表
3. 產生親證故事卡片圖片並上傳至 Google Drive

## 8. Cron Job 自動化說明

### 8.1 每週自動抽籤

系統每週一台灣時間 12:00（UTC 04:00）自動為所有未抽籤的小隊執行週任務抽籤。

**Vercel Cron 設定：**
- 路由：`GET /api/cron/auto-draw`
- Cron 表達式：`0 4 * * 1`（每週一 04:00 UTC）
- 驗證方式：請求 Header 需包含 `Authorization: Bearer [CRON_SECRET]`

**必要環境變數：**
```
CRON_SECRET=你的密鑰
```

**確認自動抽籤是否正常執行：**
- 查看「管理活動紀錄」中是否有 `auto_draw_quests` 記錄
- 記錄時間應在每週一 12:00 左右（台灣時間）

**手動觸發（測試用）：**
```bash
curl -X GET "[系統網址]/api/cron/auto-draw" \
  -H "Authorization: Bearer [CRON_SECRET]"
```

## 9. 環境變數與安全注意事項

| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 專案 URL |

| `DATABASE_URL` | PostgreSQL 直接連線字串 |
| `LINE_CHANNEL_SECRET` | LINE Bot Channel Secret |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Bot Access Token |

| `LINE_LOGIN_CHANNEL_SECRET` | LINE Login Channel Secret |
| `NEXT_PUBLIC_APP_URL` | 系統正式網址（含 https://） |

---

### 9.2 安全注意事項

1. **管理員密碼（`ADMIN_PASSWORD`）** 目前為開發用途硬編碼，**非安全邊界**，請勿用於儲存機密資料
2. **`SUPABASE_SERVICE_ROLE_KEY`** 擁有繞過 RLS 的完整資料庫存取權，嚴禁暴露至前端
3. **Cron Secret** 應使用至少 32 字元的隨機字串
4. 所有環境變數應透過 Vercel Dashboard 的 Environment Variables 管理，不應出現在程式碼或版本控制中
5. 志工密碼（VolunteerPassword）儲存在 SystemSettings 資料表，可隨時在後台更換

### 9.4 一次性修復腳本

`scripts/` 資料夾中存放資料庫修復腳本，需要時可使用 `npx ts-node scripts/<檔名>.ts` 執行。  
這些腳本為冪等操作（可重複執行），不影響正常流程。
